Making Commits: Saving Snapshots

A commit is like pressing "Save" in a video game — a permanent snapshot you can always return to.

What Is a Commit?

A commitCommitA permanent snapshot of your entire project at a specific point in time. Each commit has a unique ID, an author, a timestamp, and a message describing what changed. is a permanent snapshot of your project at a specific moment. Think of it as a save point in a video game — you can always come back to it.

Real-World Analogy

🎮 Imagine you're playing a video game. Before a boss fight, you save your game. If you lose, you reload your save point and try again. Git commits work the same way — each commit is a save point you can always return to.

Anatomy of a Commit

Good vs Bad Commit Messages

Your commit message is the only description of what changed. Future you (and your teammates) will read these to understand the project's history.

❌ Bad Messages

"fix"

"stuff"

"WIP"

"asdfghj"

"updated code"

✅ Good Messages

"Fix login button not responding on mobile"

"Add dark mode toggle to settings page"

"Remove deprecated API endpoints"

"Update README with setup instructions"

"Increase password min length to 8 chars"

A good commit message answers: "If I apply this commit, it will..." followed by a clear, specific description. Think of it like a subject line in an email.

Key Takeaways

🎉 You can now make commits like a pro!

  • A commit = a permanent snapshot of your project
  • Each commit has a unique hash, author, date, and message
  • Write clear commit messages that describe what changed
  • Use git add → git commit -m "message" to save
  • Use git log to see your commit history
