Merging: Combining Branches

Merging brings work from one branch into another. Sometimes it's smooth, sometimes there are conflicts — and that's okay!

What Is Merging?

When you're done working on a feature branch, you mergeMergeThe process of combining changes from one branch into another. Like combining two drafts of a document into one final version. it back into main to incorporate your changes.

Real-World Analogy

Imagine two people edited the same document separately. Merging is reconciling both versions into one. If they edited different paragraphs — easy! If they edited the same paragraph — you need to decide which version to keep.

The feature branch merges back into main, creating a merge commit (M)

Two Types of Merges

⏩ Fast-Forward Merge

When main hasn't changed since you branched off. Git just "fast-forwards" the main pointer. No merge commit needed.

Like: "Nobody touched main while I was away — easy!"

🔀 Three-Way Merge

When both branches have new commits. Git combines them and creates a merge commit to tie the histories together.

Like: "We both worked on things — let me combine them."

Terminal
# Step 1: Switch to the branch you want to merge INTO
git checkout main

# Step 2: Merge the feature branch into main
git merge add-dark-mode

# Step 3: Delete the branch (it's been merged)
git branch -d add-dark-mode

Merge Conflicts: When Two Edits Collide

A merge conflictMerge ConflictOccurs when two branches have edited the same line in the same file differently. Git can't decide which version to keep, so it asks you to resolve it manually. happens when two branches changed the same lines in the same file. Git can't decide which version to keep, so it asks you!

Here's what a conflict looks like in your file:

conflicted-file.txt
<<<<<<< HEAD
Welcome to our amazing website!
=======
Welcome to our beautiful new website!
>>>>>>> feature-branch

The part between <<<<< and ======= is your version (current branch). The part between ======= and >>>>> is their version (incoming branch).

🎮 Interactive: Resolve a Merge Conflict

Try resolving a conflict yourself! Pick which version to keep:

🔀 Merge Conflict Resolver
<<<<<<< HEAD (your branch)
Welcome to our amazing website!
=======
>>>>>>> feature-branch (their branch)
Welcome to our beautiful new website!

Key Takeaways

🎉 You can now merge branches and resolve conflicts!

  • Merging combines changes from one branch into another
  • Fast-forward = simple catch-up. Three-way = both branches changed
  • Merge conflicts happen when the same lines were edited differently
  • Resolve conflicts by editing the file, removing the markers, and committing
  • Conflicts are normal — don't be scared of them!
