.gitignore: Hiding Files from Git

Some files should never be tracked — secrets, build files, system junk. The .gitignore file keeps them out.

What Is .gitignore?

A .gitignore file tells Git which files and folders to completely ignore — as if they don't exist.

Real-World Analogy

🚫 Think of .gitignore as a VIP list in reverse. Everything on this list is NOT allowed into the Git club. No matter how many times you try to git add them, Git will pretend they don't exist.

What Should You Ignore?

🔑

Secrets

API keys, passwords, .env files — never commit these!

📦

Dependencies

node_modules/, venv/ — can be reinstalled

🔨

Build Files

dist/, build/, compiled files — generated from source

💻

System Files

.DS_Store, Thumbs.db — OS junk files

Example .gitignore File

.gitignore
# Dependencies
node_modules/

# Environment variables (SECRETS!)
.env
.env.local

# Build output
dist/
build/

# Log files
*.log

# OS files
.DS_Store
Thumbs.db

# IDE settings
.vscode/
.idea/

# Python
__pycache__/
*.pyc

If you accidentally commit a secret (like an API key), removing it from the file isn't enough — it's still in Git's history! You'll need to use tools like git filter-branch or BFG Repo-Cleaner to truly remove it. Prevention (using .gitignore from day one) is much easier than the cure.

🎉 You now know how to keep your repo clean! Always create a .gitignore file as the first thing in any new project.
