Clone vs Fork

Two ways to get someone else's code — one downloads it, the other gives you your own copy to modify.

Clone: Photocopying a Book

git clone downloads a full copy of a repo to your computer — all files, all history, everything.

Real-World Analogy

📖 Clone = photocopying a library book. You get an exact copy to read and study, but the original stays at the library. You can't write in the library's book.

Terminal
git clone https://github.com/someone/cool-project.git

Fork: Buying Your Own Copy

A forkForkA personal copy of someone else's repository on YOUR GitHub account. You can modify it freely without affecting the original. creates your own copy of someone's repo on your GitHub account. You can then modify it freely.

Real-World Analogy

📝 Fork = buying your own copy of a book so you can write notes in the margins, highlight passages, and even add new chapters. The original book is unchanged.

Side-by-Side Comparison

CloneFork
What it doesDownloads to your computerCopies to your GitHub account
Where it livesYour laptopYour GitHub, then your laptop
Can you push?Only if you have permissionYes, to YOUR fork
Can you contribute back?Only if you're a collaboratorYes, via Pull Request
Use caseWorking on your own repoContributing to others' repos
