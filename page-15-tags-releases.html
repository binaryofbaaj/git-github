Tags & Releases

Tags mark important milestones in your project — like version numbers on a software box.

What Are Tags?

A tagTagA permanent label on a specific commit — like a bookmark. Usually used for version numbers like v1.0, v2.3.1. is a permanent label on a specific commit — like a sticky note that says "this is version 1.0!"

Real-World Analogy

🏷️ Tags are like sticky notes on specific chapters of a book — "this is the version we shipped!" or "this is the one the client approved."

Semantic Versioning

Most projects use semantic versioning: MAJOR.MINOR.PATCH

MAJOR (v2.0.0)

Breaking changes. Existing features may stop working. Like a major redesign.

MINOR (v2.1.0)

New features added. Everything old still works. Like adding a new room to a house.

PATCH (v2.1.3)

Bug fixes only. No new features, just fixing problems. Like patching a hole.

Commands

git tag v1.0.0
git push origin v1.0.0

🎉 You now understand tags and releases! Use them to mark important versions of your project.
