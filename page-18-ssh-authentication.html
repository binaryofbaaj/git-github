SSH Keys & Authentication

Connect to GitHub securely without typing your password every time. Like having a digital passport.

HTTPS vs SSH

There are two ways to connect your computer to GitHub:

🔗 HTTPS

Simpler to set up. Uses personal access tokens. Good for getting started.

https://github.com/user/repo.git

🔐 SSH

More secure. No password needed after setup. The professional choice.

git@github.com:user/repo.git

SSH Keys: The Lock & Key Analogy

Real-World Analogy

🔑 An SSH key pair works like a house and its key. Your public key = your house address (you can share it with everyone — including GitHub). Your private key = your house key (you NEVER share this with anyone). When you push code, GitHub checks if your private key matches the public key on file.

Setting Up SSH

Terminal
# Step 1: Generate a key pair
ssh-keygen -t ed25519 -C "you@example.com"

# Step 2: Start the SSH agent
eval "$(ssh-agent -s)"

# Step 3: Add your key to the agent
ssh-add ~/.ssh/id_ed25519

# Step 4: Copy your PUBLIC key
cat ~/.ssh/id_ed25519.pub
# Copy the output → GitHub → Settings → SSH Keys → New

# Step 5: Test the connection
ssh -T git@github.com

Never share your private key (the file without .pub). Only share the public key (the file ending in .pub). If someone gets your private key, they can impersonate you!

🎉 You're now connected securely! SSH means no more typing passwords — push and pull seamlessly.
