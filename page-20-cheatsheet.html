Interactive Git Cheat Sheet

Your ultimate command reference card. Search in real-time, click any command to copy, or print/save as PDF.

1. Setup & Configuration

Configure your identity, default preferences, and terminal visuals.

git config --global user.name "John Doe"
Set the name associated with all your commit transactions.
👤 Safe
git config --global user.email "john@example.com"
Set the email associated with all your commit transactions.
✉️ Safe
git config --global init.defaultBranch main
Set the default branch name to 'main' for new repositories.
🌿 Safe
git config --global color.ui auto
Enable helpful colorization of command-line output.
🎨 Safe
git config --list
Show all Git configuration settings currently active on your system.
⚙️ Safe
git config --global --edit
Open your global configuration file directly in a text editor.
✏️ Caution

2. Starting a Project

Initialize new repositories or clone existing ones from remotes.

git init
Initialize a new, blank local Git repository in the current directory.
📁 Safe
git clone [url]
Download a repository from GitHub to your local machine.
📥 Safe
git remote -v
List all remote repositories linked to your local repository.
🌐 Safe
git remote add origin [url]
Associate a remote GitHub URL with the shortcut name 'origin'.
➕ Safe
git remote rename [old] [new]
Rename a remote repository shortcut name (e.g. master to main-remote).
✏️ Safe
git remote remove [name]
Disconnect your local repository from a remote configuration.
❌ Safe

3. Staging & Committing

Pack changes into the staging area and save them permanently to repository history.

git status
Show the state of modified/untracked files in the working directory.
🔍 Safe
git add [file]
Add a specific file to the staging area (preparing it for a commit).
📄 Safe
git add .
Add all new, modified, and deleted files in the directory to staging.
📁 Safe
git add -p
Interactively review and select specific parts (hunks) of files to stage.
✂️ Safe
git commit -m "message"
Save staged changes as a new project snapshot with a description.
💾 Safe
git commit -am "message"
Automatically stage all tracked, modified files and commit them.
⚡ Safe
git commit --amend
Combine staged changes with the last commit to fix mistakes or update messages.
✏️ Caution

4. Branching

Work in parallel timelines to develop features or test bugs safely.

git branch
List all local branches. The active branch is highlighted.
🌿 Safe
git branch -r
List all remote branches tracked in your repository.
🌐 Safe
git branch -a
List all local and remote branches in one view.
🌿 Safe
git branch [branch-name]
Create a new branch from your current location, but do not switch to it.
➕ Safe
git checkout [branch]
Switch your working directory to the specified branch.
🔄 Safe
git switch [branch]
The modern, dedicated command to switch between branches.
🔄 Safe
git checkout -b [branch]
Create a new branch and switch to it immediately in one command.
🌿 Safe
git switch -c [branch]
The modern way to create a branch and switch to it immediately.
🌿 Safe
git branch -d [branch]
Delete a branch safely. Git warns if the branch has unmerged work.
🗑️ Caution
git branch -D [branch]
Force delete a branch, ignoring merge safety checks. Can lose work!
⚠️ Destructive

5. Merging & Rebase

Integrate branches together or resolve conflicts during code integration.

git merge [branch]
Merge commits from the target branch into your active branch.
🔀 Caution
git merge --abort
Stop a merge conflict state and restore files to pre-merge state.
🛑 Safe
git merge --continue
Finalize a merge commit after resolving any file conflicts.
▶️ Safe
git rebase [branch]
Re-apply commits from your current branch on top of another branch.
🔄 Caution
git rebase --abort
Abort an active rebase and return your branch to its original state.
🛑 Safe

6. Collaboration & Remote Syncing

Upload, download, and synchronize code with GitHub or other servers.

git fetch
Download new history, branches, and tags from remote without merging them.
📥 Safe
git pull
Fetch remote changes and immediately merge them into your local branch.
📥 Caution
git push [remote] [branch]
Upload your local commits to the remote repository.
📤 Safe
git push -u origin [branch]
Push your branch and link it to the remote branch for simple future pulls/pushes.
📌 Safe
git push --force
Force upload commits, overwriting remote history. Highly dangerous in teams!
⚠️ Destructive
git push --force-with-lease
Safer force push. Stops if someone else has pushed new remote commits.
🛡️ Caution

7. Inspecting History

Explore the logs, compare differences, and inspect commit changes.

git log
Show the chronological list of commits for the active branch.
📜 Safe
git log --oneline
Display commit logs in a compact, single-line format.
📏 Safe
git log --graph --oneline --all
Show an ASCII graphical branch structure alongside commit history.
📊 Safe
git diff
Show unstaged changes in files that have not yet been added.
🔍 Safe
git diff --staged
Show differences between staged files and the last committed files.
🔍 Safe
git diff [branch1] [branch2]
Compare and show all changes between branch1 and branch2.
🔍 Safe
git show [commit]
Display metadata, message, and changes made in a specific commit hash.
👀 Safe
git blame [file]
Display line-by-line which commit and author changed each part of a file.
👤 Safe

8. Undoing & Restoring

Correct mistakes, revert historical commits, or clear your working folder.

git restore [file]
Discard local edits, restoring the file to the last committed version.
⏪ Caution
git restore --staged [file]
Unstage a file, keeping its changes in the working directory.
📤 Safe
git reset HEAD [file]
Unstage a file (an older legacy equivalent of git restore --staged).
⏪ Safe
git reset --soft HEAD~1
Undo the last commit. Changes are kept in the staging area.
⏪ Caution
git reset --mixed HEAD~1
Undo the last commit and unstage files. Edits are kept in working directory.
⏪ Caution
git reset --hard HEAD~1
Undo the last commit and delete all file edits. Destroyed changes are unrecoverable!
💥 Destructive
git revert [commit]
Create a new commit that applies opposite edits to safely undo a previous commit.
🛡️ Safe
git clean -df
Permanently delete all untracked files and folders in your repository.
🧹 Destructive

9. Stashing Changes

Temporarily shell away unfinished changes to clean your workspace.

git stash
Store all tracked, uncommitted changes and revert files to last commit state.
📦 Safe
git stash save "message"
Save a stash with a custom text tag to easily identify it.
🏷️ Safe
git stash list
Show all stashes currently stored in your repository.
📋 Safe
git stash pop
Restore the latest stashed changes and remove them from stash storage.
📤 Safe
git stash apply
Restore the latest stashed changes but keep them saved in stash storage.
🔄 Safe
git stash drop stash@{0}
Permanently discard a specific stash from the stash storage list.
🗑️ Destructive
git stash clear
Permanently delete all stashed snapshots from the repository storage.
⚠️ Destructive

10. Tagging & Releases

Tag milestones, specific release versions, and publish versions.

git tag
List all historical tags/releases in the local repository.
🏷️ Safe
git tag -a v1.0.0 -m "message"
Create a new local annotated tag at the current commit.
📌 Safe
git tag -d v1.0.0
Delete a version tag locally from your computer.
🗑️ Safe
git push origin v1.0.0
Push a specific version tag to the remote repository (GitHub).
📤 Safe
git push origin --tags
Push all local release version tags to origin remote.
📤 Safe
git push origin --delete v1.0.0
Delete a release version tag from origin remote.
🗑️ Caution
