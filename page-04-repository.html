What Is a Repository?

A repository (or "repo") is simply a folder that Git is watching and tracking. Let's explore what that really means.

The Simple Definition

A repositoryRepository (Repo)A project folder that Git is tracking. It contains your files plus a hidden .git folder that stores all of Git's data about changes and history. is just a folder — but one that Git is actively watching. Every change you make to files inside this folder gets noticed by Git.

Real-World Analogy

Think of a regular folder as an ordinary room. A Git repository is a room with security cameras installed. The room itself doesn't change — but now every movement inside it is being recorded. You can review the footage (history) anytime.

Creating a Repository with git init

Local vs Remote Repositories

There are two types of repos you'll work with:

📁 Local Repository

Lives on your computer. This is where you do your actual work — writing code, creating files, making changes.

🏠 Analogy: Your desk drawer

☁️ Remote Repository

Lives on a server (like GitHub). It's an online copy of your project that others can access.

🗄️ Analogy: A shared filing cabinet in the office

Inside the .git Folder

When you run git init, Git creates a hidden folder called .git inside your project. This is Git's brain — it stores everything Git needs to track your project. Let's peek inside:

📁 .git/
📁 objects/ — Stores all file versions and commits
📁 refs/ — Stores branch and tag pointers
📄 HEAD — Points to your current branch
📄 config — Repo-specific settings
📄 index — The staging area

Never manually edit or delete the .git folder! If you do, Git will lose all its memory of your project's history. It's like erasing the security camera footage.

Key Takeaways

🎉 You now understand repositories!

  • A repo is a folder Git is tracking
  • git init turns any folder into a repo by creating a .git folder
  • Local repos live on your computer, remote repos live online (GitHub)
  • The .git folder is Git's brain — never delete it!
