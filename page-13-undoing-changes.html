Undoing Mistakes: The Safety Net

Git gives you multiple levels of "undo" — from a simple Ctrl+Z to full time travel. Here's when to use each one.

Decision Tree: What Do I Want to Undo?

The Commands

Terminal
# Undo changes in working directory
git restore filename.txt

# Unstage a file
git restore --staged filename.txt

# Temporarily save work
git stash
git stash pop   # Get it back

# Safely undo a commit (creates new "undo" commit)
git revert abc1234

# Roll back commits (LOCAL/unshared only!)
git reset --soft HEAD~1   # Keep staged
git reset --mixed HEAD~1  # Keep unstaged
git reset --hard HEAD~1   # ⚠️ Delete everything!

git reset --hard permanently deletes your changes. Only use it for local, unshared work. For shared branches, always use git revert instead.

Real-World Analogy

🧳 git stash is like putting your unfinished work into a secret pocket. You can go do something else, and then pull your work back out of the pocket when you're ready. Super handy when you need to switch branches quickly!

🎉 You now have a complete safety net! No matter what goes wrong, Git has an undo command for it.
