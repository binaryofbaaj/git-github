Remote Repositories

Connect your local work to GitHub — upload with push, download with pull, and collaborate with the world.

What Is a Remote?

A remoteRemoteA version of your repository stored on a server (like GitHub). The default remote is usually named "origin". is a copy of your repo stored on a server — usually GitHub. The default remote is called origin.

Real-World Analogy

📮 Think of it like a post office system. Your local repo is your home desk where you write letters. The remote repo is the post office. git push = mailing letters to the post office. git pull = picking up letters others have sent.

push uploads, pull downloads, clone copies the entire repo for the first time

Essential Remote Commands

Terminal
# Connect your local repo to GitHub
git remote add origin https://github.com/you/my-project.git

# Push for the first time (-u sets up tracking)
git push -u origin main

# Push future commits (simpler after first time)
git push

# Get latest changes from GitHub
git pull

# See your remotes
git remote -v

The -u flag in git push -u origin main sets up a "tracking" connection. After this first push, you can just type git push and Git knows where to send it!

Key Takeaways

🎉 You can now work with remote repositories!

  • Remote = an online copy of your repo (usually on GitHub)
  • git push uploads your commits to GitHub
  • git pull downloads the latest changes
  • origin is the default name for your remote
  • git remote add origin URL connects your local repo to GitHub
