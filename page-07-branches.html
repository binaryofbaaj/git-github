Branches: Parallel Universes

Branches let you create alternate timelines for your project — experiment freely without breaking anything.

What Is a Branch?

A branchBranchA separate line of development. Like an alternate timeline where you can make changes without affecting the original (main) timeline. is a separate copy of your project's timeline where you can make changes without affecting the main version.

Real-World Analogy

🌳 Think of a tree. The trunk is your main branch — the stable, official version. Branches grow off the trunk in different directions, each exploring something different. The roots (history) are shared, but each branch can grow independently.

The main branch is the trunk. Feature branches grow separately, each with their own commits.

Branch Commands

Terminal
# Create a new branch
git branch add-dark-mode

# Switch to it
git checkout add-dark-mode
# OR the modern way:
git switch add-dark-mode

# Create AND switch in one command (most common)
git checkout -b add-dark-mode

# See all branches (* = current)
git branch

# Delete a branch (after merging)
git branch -d add-dark-mode

The HEADHEADA pointer that tells Git which branch (and which commit) you're currently working on. Think of it as a "You Are Here" marker on a map. pointer tells Git which branch you're on. When you switch branches, HEAD moves to point at the new branch.

Key Takeaways

🎉 You now understand branches!

  • A branch is a separate timeline for your project
  • main is the default, official branch
  • git checkout -b name creates and switches to a new branch
  • HEAD points to your current branch — it's your "You Are Here" marker
  • Branches are cheap and fast — use them freely!
